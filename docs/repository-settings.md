# Repository Settings

Use this checklist after pushing the monorepo to GitHub.

## GitHub Repository

- Default branch: `main`.
- Actions: enabled.
- CI workflow: `.github/workflows/ci.yml`.
- Package manager: npm, pinned in root `package.json`.
- Validation command: `npm run build`.
- Dependabot: root, apps, and `packages/ui` npm metadata.

## Cloudflare Pages Projects

Create three Cloudflare Pages projects from the same GitHub monorepo.

| Project | Root directory | Build command | Output directory |
| --- | --- | --- | --- |
| `rehabtrainerhub` | `/` | `npm ci --workspaces=false && npm run build:hub` | `apps/rehabtrainerhub/out` |
| `stroketrainer` | `/` | `npm ci --workspaces=false && npm run build:stroke` | `apps/stroketrainer/dist` |
| `visiontrainer` | `/` | `npm ci --workspaces=false && npm run build:vision` | `apps/visiontrainer/dist` |

Set Node.js compatibility to Node 24 or newer.

## Environment Variables

Hub:

- `NEXT_PUBLIC_SITE_URL=https://your-domain.example`

StrokeTrainer optional external Vosk model URLs:

- `VITE_VOSK_MODEL_ZH_URL`
- `VITE_VOSK_MODEL_EN_URL`
- `VITE_VOSK_MODEL_ZH_VOCAB_URL`
- `VITE_VOSK_MODEL_EN_VOCAB_URL`
- `VITE_VOSK_MODEL_MIN_BYTES`

Use repository or Pages variables for public model URLs. Use secrets only for
private credentials such as Cloudflare API tokens or Discord webhook values.

## Custom Domains

Recommended mapping:

- `rehabhub.tw` -> `rehabtrainerhub`
- `stroke.rehabhub.tw` or `/StrokeTrainer/` -> `stroketrainer`
- `vision.rehabhub.tw` or `/VisionTrainer/` -> `visiontrainer`

The Hub currently links to `/StrokeTrainer/` and `/VisionTrainer/` for a
single-domain subdirectory deployment. If using subdomains, update
`apps/rehabtrainerhub/app/page.tsx` and `GlobalPortalLinks` props accordingly.

## SEO

- Next.js metadata is defined in `apps/rehabtrainerhub/app/layout.tsx`.
- `robots.txt` is generated by `apps/rehabtrainerhub/app/robots.ts`.
- `sitemap.xml` is generated by `apps/rehabtrainerhub/app/sitemap.ts`.
- Submit the production sitemap to Google Search Console after domain setup.

## Redirects And Archive

- Configure Cloudflare Bulk Redirects from old `*.pages.dev` URLs to the new
  production paths or subdomains.
- Update the old `StrokeTrainer` and `VisionTrainer` README files with the new
  monorepo and production URLs.
- Archive the old repositories after verifying traffic has moved.
- Remove nested `apps/stroketrainer/.git` and `apps/visiontrainer/.git`
  directories before the first root monorepo commit so the apps are tracked as
  ordinary folders.
